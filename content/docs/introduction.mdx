## JaguarClaw 是什么？

JaguarClaw 是一个**企业级桌面 AI 助手**。它不只是聊天框 —— 它是一个真正的 Agent，能够：

- 🧠 **思考** — 理解任务，拆解步骤
- 🛠️ **执行** — 操作文件、运行命令、搜索网络
- 💾 **记忆** — 跨会话保存重要信息

它运行在你的机器上，数据永不离开你的网络。

## 为什么选择 JaguarClaw？

### 不是聊天框，是 Agent

大多数 AI 助手只是聊天框。你问，它答，就这样。

JaguarClaw 不同。给它一个任务，它会：

1. **思考**需要做什么
2. **执行**相应的工具
3. **观察**执行结果
4. **迭代**直到任务完成

```
用户: "分析今天的错误日志，生成摘要报告"

  Think  → 我需要先找到今天的日志文件
  Act    → read_file("/var/log/app/2026-03-05.log")
  Observe → [2546 行日志内容...]
  Think  → 我发现 3 类重复错误，让我分类统计
  Act    → write_file("report.md", "# 错误摘要\n...")
  Done   → "我已创建 report.md，包含 3 类错误的详细分析"
```

### 安全可控

- **本地运行** — 代码在你机器上执行
- **危险操作确认** — `rm -rf`、`DROP TABLE` 等需要你批准
- **透明日志** — 每一步都有记录，可审计

### 灵活接入

支持任意 OpenAI 兼容的 LLM：

## 核心功能

### ReAct 引擎

实现 Reasoning + Acting 循环：

- 多步推理，可配置最大步数和超时
- 流式 token 输出，实时显示思考过程
- 任意步骤可取消

### 工具系统

内置 10+ 安全工具：

| 工具 | 功能 | 安全级别 |
|------|------|----------|
| `read_file` | 读取文件 | 安全 |
| `write_file` | 写入文件 | 安全 |
| `shell` | 执行命令 | 危险命令需确认 |
| `web_search` | 网页搜索 | 安全 |
| `memory_search` | 搜索记忆 | 安全 |
| `memory_save` | 保存记忆 | 安全 |
| `sessions_spawn` | 启动子代理 | 安全 |

### 技能系统

技能是模块化的专家行为，定义为 Markdown 文件：

```yaml
---
name: code-review
description: 审查代码中的 bug、安全问题和风格问题
allowed-tools:
  - read_file
  - web_search
---

# 代码审查指南
审查代码时，重点关注...
```

- 兼容 Claude Skills 格式
- 热加载 — 添加技能文件，立即可用
- 按技能限制工具访问

### 长期记忆

跨会话记忆，以 Markdown 为存储格式：

- 全文搜索
- 语义搜索（可选 pgvector）
- 时间衰减排序 — 最近的信息优先

### 子代理并行执行

复杂任务可以并行处理：

```
用户: "对比我们 3 个 API 端点的性能"

  主代理 → 启动 3 个子代理
    子代理 1 → 压测 /api/users
    子代理 2 → 压测 /api/orders
    子代理 3 → 压测 /api/products
  主代理 → 汇总结果为对比表格
```

### MCP 协议支持

连接任何 [Model Context Protocol](https://modelcontextprotocol.io/) 服务器：

- **传输方式**：STDIO、SSE、HTTP
- **官方服务器**：filesystem、fetch、git、postgres
- **第三方**：GitHub、Slack、AWS、Kubernetes

### 定时任务

使用 cron 表达式设置周期任务：

- 持久化任务存储
- **交付渠道**：Email (SMTP)、Webhook (HTTP POST)
- 每次运行独立会话上下文
